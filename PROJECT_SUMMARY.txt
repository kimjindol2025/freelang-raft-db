================================================================================
ARCHITECTURE
================================================================================

4-Layer Design:

  LAYER 4: Database Operations
           Get, Set, Delete, Exists
           ↓
  LAYER 3: Consensus Protocol
           Leader Election, Log Replication
           ↓
  LAYER 2: State Machine
           Entry Application, Commit Index
           ↓
  LAYER 1: Node State
           Term Management, Log Management, Voting

Key Components:
  - RaftNode: Core node state with currentTerm, logs, votedFor
  - RaftConsensus: Consensus state combining node + election state
  - DistributedDB: Database operations with state machine
  - StateMachine: Key-value store for committed entries

Data Flow:
  Database Op → Append to Log → Leader Replicates → Quorum Acks
  → Commit Index Advances → Entry Applied to State Machine
